# CARLA XOSC Generator Guide

The CARLA adapter converts expanded core scenario YAML files into CARLA-compatible
OpenSCENARIO `.xosc` files.

## Scenario Types And CLI Selector

The adapter reads core YAML and writes XOSC inside the same
`scenarios/<scenario_type>/` root. Its selector optionally starts with the
scenario type:

```text
[<scenario_type>/]<feature_domain>/<functional>/<scenario_id>
[<scenario_type>/]<feature_domain>/<functional>
```

Selectors without the prefix continue to address `general_scenarios`.
Specify `ncap_scenarios` to generate Euro NCAP cases:

```powershell
python adapters\carla\longitudinal_feature\generated.py longitudinal/acc/acc_csc_001 --clean
python adapters\carla\longitudinal_feature\generated.py general_scenarios/longitudinal/acc/acc_csc_001 --clean
python adapters\carla\brake_feature\generated.py ncap_scenarios/brake/aeb/aeb_cpna_001 --clean
python adapters\carla\brake_feature\generated.py ncap_scenarios/brake/aeb --clean
python adapters\carla\brake_feature\generated.py ncap_scenarios --all --clean
```

For an NCAP core whose logic contains:

```yaml
parametersDeclaration:
  - type: CPNA
```

the generator renders and inserts:

```text
scenarios/ncap_scenarios/templates/ParameterDeclarations/<domain>/<function>/CPNA_parameters.xosc
```

before inserting the maneuver block. CPNA derived values are materialized
during generation because the bundled ScenarioRunner accepts `$parameter`
references but does not evaluate arithmetic expressions in declarations.

For CPNA, fixed protocol values such as `ev_initS`, `ev_initTTC`,
`VRU_finalSpeed_kph` and `VRU_initLatDist` are sourced only from
`CPNA_parameters.xosc`. Case inputs such as `overlap` and `ev_speed_kph` are
read from the core YAML. Derived values including `_VRU_initS` are recalculated
from those sources on every generation; do not edit generated XOSC files.
For CPNA, the VRU trajectory always starts from the configured nearside
(`VRU_trajectoryOrientation`); changing `overlap` changes only the impact
point. With the current positive-nearside configuration, `25%` maps to
`+0.4625 m` and `75%` maps to `-0.4625 m` for an EV width of `1.85 m`.

The GUI batch runner regenerates each selected XOSC from its current core YAML
and template immediately before starting the CARLA case.

Input and output for any selected type:

```text
scenarios/<scenario_type>/core/<feature_domain>/<functional>/<scenario_id>/<case_id>.yaml
=> scenarios/<scenario_type>/generated/carla/<feature_domain>/<functional>/<scenario_id>/<case_id>.xosc
```

Example:

```text
scenarios/general_scenarios/core/longitudinal_feature/ACC/acc_csc_001/acc_csc_001_001.yaml

=> scenarios/general_scenarios/generated/carla/longitudinal_feature/ACC/acc_csc_001/acc_csc_001_001.xosc
```

## 1. Generator Structure

The generator is split by `feature_domain` to keep each domain maintainable:

```text
adapters/carla/
|-- _generated_common.py              # Shared generation engine
|-- generated_wrapper.py              # Compatibility wrapper for all domains
|-- longitudinal_feature/
|   `-- generated.py                  # Longitudinal-only wrapper
|-- lateral_feature/
|   `-- generated.py                  # Lateral-only wrapper
|-- brake_feature/
|   `-- generated.py                  # Brake-only wrapper
`-- parking_feature/
```

Domain wrappers only accept selectors for their own domain:

```powershell
python adapters\carla\longitudinal_feature\generated.py longitudinal/acc/acc_csc_001 --clean
python adapters\carla\lateral_feature\generated.py lateral/lka/lka_csc_001 --clean
python adapters\carla\brake_feature\generated.py brake/aeb/aeb_csc_001 --clean
```

The compatibility wrapper still supports the old command style:

```powershell
python adapters\carla\generated_wrapper.py longitudinal/acc/acc_csc_001 --clean
python adapters\carla\generated_wrapper.py brake/aeb/aeb_csc_001 --clean
```

## 2. Required Metadata

Each core YAML is usually generated by `expander.py` and contains the original
logical YAML under `logic`.

The metadata must exist either at the top level or under `logic`:

```yaml
logic:
  functional: ACC
  feature_domain: Longitudinal
```

The generator uses these keys to resolve:

1. Storyboard template.
2. Maneuver block folder.
3. Controller FMU files.
4. Output folder.

Supported `feature_domain` values currently map like this:

```text
Longitudinal -> longitudinal_feature
Lateral      -> lateral_feature
Parking      -> parking_feature
Brake        -> brake_feature
```

## 3. Required Templates

The paths below illustrate `general_scenarios`. For another scenario type,
replace that root with the selected one, for example `ncap_scenarios`.

For a core YAML with:

```yaml
functional: ACC
feature_domain: Longitudinal
```

The generator expects storyboard templates in:

```text
scenarios/general_scenarios/templates/storyboard/longitudinal_feature/ACC/
```

For a core YAML with:

```yaml
functional: AEB
feature_domain: Brake
```

The generator expects storyboard templates in:

```text
scenarios/general_scenarios/templates/storyboard/brake_feature/AEB/
```

Single-TV and multi-TVs storyboard file names follow this convention:

```text
<FUNCTIONAL>_single-TV_storyboard.xosc
<FUNCTIONAL>_multi-TVs_storyboard.xosc
```

Examples:

```text
ACC_single-TV_storyboard.xosc
ACC_multi-TVs_storyboard.xosc
AEB_single-TV_storyboard.xosc
AEB_multi-TVs_storyboard.xosc
```

## 4. Maneuver Blocks

Maneuver blocks are also resolved by domain/function:

```text
scenarios/general_scenarios/templates/maneuver_blocks/<feature_domain>/<functional>/
```

Examples:

```text
scenarios/general_scenarios/templates/maneuver_blocks/longitudinal_feature/ACC/
scenarios/general_scenarios/templates/maneuver_blocks/brake_feature/AEB/
```

There is no large hard-coded maneuver map in each domain `generated.py`.
`_generated_common.py` scans the `.xosc` files in the selected domain/function
block folder and discovers maneuver types from `<Event name="...">`.

Example block:

```xml
<Event name="${actor}_brake_event" priority="overwrite">
```

This is discovered as:

```text
brake_event
```

The file stem is also registered as `<stem>_event`, so:

```text
emergency_brake.xosc -> emergency_brake_event
```

## 5. Selector Format

Use this selector format. The scenario-type prefix is optional only for
`general_scenarios`:

```text
[<scenario_type>/]<feature_domain>/<functional>/<scenario_id>
```

Examples:

```text
longitudinal/acc/acc_csc_001
brake/aeb/aeb_csc_001
ncap_scenarios/brake/aeb/aeb_cpna_001
```

The selector is case-insensitive for domain and functional names:

```powershell
python adapters\carla\longitudinal_feature\generated.py Longitudinal/ACC/acc_csc_001
python adapters\carla\lateral_feature\generated.py Lateral/LKA/lka_csc_001
python adapters\carla\brake_feature\generated.py Brake/AEB/aeb_csc_001
python adapters\carla\brake_feature\generated.py ncap_scenarios/Brake/AEB/aeb_cpna_001
```

## 6. Generate One Scenario Folder

Use this after `expander.py` has generated the matching core YAML files.

Longitudinal:

```powershell
python adapters\carla\longitudinal_feature\generated.py longitudinal/acc/acc_csc_001 --clean
```

Brake:

```powershell
python adapters\carla\brake_feature\generated.py brake/aeb/aeb_csc_001 --clean
```

NCAP brake:

```powershell
python adapters\carla\brake_feature\generated.py ncap_scenarios/brake/aeb/aeb_cpna_001 --clean
```

```powershell
python adapters\carla\generated_wrapper.py brake/aeb/aeb_csc_001 --clean
```

## 7. Generate Multiple Specific Scenarios

Pass multiple selectors in one command:

```powershell
python adapters\carla\longitudinal_feature\generated.py longitudinal/acc/acc_csc_001 longitudinal/acc/acc_csc_002 --clean
```

## 8. Generate A Range

Use a selector prefix plus `--from` and `--to`.

```powershell
python adapters\carla\longitudinal_feature\generated.py longitudinal/acc/acc_csc --from 1 --to 22 --clean
```

Rules:

1. `--from` and `--to` must be used together.
2. Range mode accepts exactly one selector prefix.
3. Missing core scenario folders are skipped and printed as `[SKIP]`.

## 9. Generate A Whole Functional Folder

Use only `<feature_domain>/<functional>`:

```powershell
python adapters\carla\longitudinal_feature\generated.py longitudinal/acc --clean
python adapters\carla\lateral_feature\generated.py lateral/lka --clean
python adapters\carla\brake_feature\generated.py brake/aeb --clean
```

## 10. Generate Everything In A Domain

Domain-specific `--all` only scans that domain:

```powershell
python adapters\carla\longitudinal_feature\generated.py --all --clean
python adapters\carla\lateral_feature\generated.py --all --clean
python adapters\carla\brake_feature\generated.py --all --clean
```

The compatibility wrapper scans all domains:

```powershell
python adapters\carla\generated_wrapper.py --all --clean
```

To generate all cases for another scenario type, prefix `--all` with the type:

```powershell
python adapters\carla\brake_feature\generated.py ncap_scenarios --all --clean
```

```text

```

## 12. Common Errors

`Core scenario folder not found`

The core YAML folder does not exist yet. Run `expander.py` first.

`Selector ... belongs to ... but this generator only supports ...`

You used a selector with the wrong domain wrapper. For example, run AEB with:

```powershell
python adapters\carla\brake_feature\generated.py brake/aeb/aeb_csc_001 --clean
```

`Storyboard template not found`

Check that the storyboard template exists:

```text
scenarios/general_scenarios/templates/storyboard/<feature_domain>/<functional>/
```

`Maneuver block directory not found`

Check that the maneuver block folder exists:

```text
scenarios/general_scenarios/templates/maneuver_blocks/<feature_domain>/<functional>/
```

`Unknown maneuver type`

The logical YAML contains a maneuver `type` that was not discovered from any
`<Event name="...">` or block filename in that domain/function maneuver block
folder.

## 13. Recommended Workflow

1. Run `expander.py` for one scenario.
2. Inspect the generated core YAML file.
3. Run the domain-specific `generated.py` for that scenario.
4. Inspect the generated `.xosc` file.
5. Run a range or whole functional folder only after the single scenario is correct.
