# CARLA XOSC Generator Guide

`generated.py` converts core scenario YAML files into CARLA-compatible OpenSCENARIO `.xosc` files.

Input:

```text
scenarios/general_scenarios/core/<feature_domain>/<functional>/<scenario_id>/<case_id>.yaml
```

Output:

```text
scenarios/general_scenarios/generated/carla/<feature_domain>/<functional>/<scenario_id>/<case_id>.xosc
```

Example:

```text
scenarios/general_scenarios/core/longitudinal_feature/ACC/acc_csc_001/acc_csc_001_001.yaml

=> scenarios/general_scenarios/generated/carla/longitudinal_feature/ACC/acc_csc_001/acc_csc_001_001.xosc
```

## 1. Required Metadata

Each core YAML is usually generated by `expander.py` and contains the original logical YAML under `logic`.

The metadata must exist either at the top level or under `logic`:

```yaml
logic:
  functional: ACC
  feature_domain: Longitudinal
```

`generated.py` uses these keys to resolve:

1. The storyboard template.
2. The maneuver block folder.
3. The output folder.

Supported `feature_domain` values currently map like this:

```text
Longitudinal -> longitudinal_feature
Lateral      -> lateral_feature
Parking      -> parking_feature
Brake        -> brake_feature
```

## 2. Required Templates

For a core YAML with:

```yaml
functional: ACC
feature_domain: Longitudinal
```

The expected storyboard template is:

```text
scenarios/general_scenarios/templates/storyboard/longitudinal_feature/ACC_storyboard.xosc
```

The expected maneuver blocks are in:

```text
scenarios/general_scenarios/templates/maneuver_blocks/longitudinal_feature/ACC/
```

Each maneuver type in the logical YAML must exist in the internal maneuver map in `generated.py`.

Example:

```yaml
maneuvers:
  - type: cruise_event
    actor: ev
```

This uses:

```text
scenarios/general_scenarios/templates/maneuver_blocks/longitudinal_feature/ACC/cruise_control.xosc
```

## 4. Selector Format

Use this selector format:

```text
<feature_domain>/<functional>/<scenario_id>
```

Example:

```text
longitudinal/acc/acc_csc_001
```

The selector is case-insensitive for domain and functional names:

```powershell
python generated.py longitudinal/acc/acc_csc_001
python generated.py Longitudinal/ACC/acc_csc_001
```

Both resolve to:

```text
scenarios/general_scenarios/core/longitudinal_feature/ACC/acc_csc_001/
```

## 5. Generate One Scenario Folder

Use this after `expander.py` has generated the matching core YAML files.

```powershell
python generated.py longitudinal/acc/acc_csc_001
```

Expected output:

```text
scenarios/general_scenarios/generated/carla/longitudinal_feature/ACC/acc_csc_001/
```

## 6. Generate One Scenario With Clean

Use `--clean` when you want to remove old `.xosc` files in the output folder before regenerating.

```powershell
python generated.py longitudinal/acc/acc_csc_001 --clean
```

Use this when the core YAML or templates changed and you want the generated `.xosc` folder to match the current source exactly.

## 7. Generate Multiple Specific Scenarios

Pass multiple selectors in one command:

```powershell
python generated.py longitudinal/acc/acc_csc_001 longitudinal/acc/acc_csc_002 longitudinal/acc/acc_csc_003
```

With clean:

```powershell
python generated.py longitudinal/acc/acc_csc_001 longitudinal/acc/acc_csc_002 --clean
```

## 8. Generate A Range

Use a selector prefix plus `--from` and `--to`.

```powershell
python generated.py longitudinal/acc/acc_csc --from 1 --to 22
```

This generates `.xosc` files for:

```text
acc_csc_001
acc_csc_002
...
acc_csc_022
```

With clean:

```powershell
python generated.py longitudinal/acc/acc_csc --from 1 --to 22 --clean
```

Rules:

1. `--from` and `--to` must be used together.
2. Range mode accepts exactly one selector prefix.
3. Missing core scenario folders are skipped and printed as `[SKIP]`.

## 9. Generate A Whole Functional Folder

Use only `<feature_domain>/<functional>`:

```powershell
python generated.py longitudinal/acc
```

This generates every core scenario folder under:

```text
scenarios/general_scenarios/core/longitudinal_feature/ACC/
```

With clean:

```powershell
python generated.py longitudinal/acc --clean
```

## 10. Generate Everything

Use `--all` to generate `.xosc` files from every core scenario folder under `scenarios/general_scenarios/core/`.

```powershell
python generated.py --all
```

With clean:

```powershell
python generated.py --all --clean
```

Use this carefully because it may regenerate many `.xosc` folders.

## 11. End-To-End Example

From repo root:

```powershell
python expander\expander.py longitudinal/acc/acc_csc_001 --clean
python adapters\carla\generated.py longitudinal/acc/acc_csc_001 --clean
```

Expected flow:

```text
logical YAML + parameter YAML
=> core YAML files
=> CARLA .xosc files
```

Final output:

```text
scenarios/general_scenarios/generated/carla/longitudinal_feature/ACC/acc_csc_001/*.xosc
```

## 12. Common Errors

`Core scenario folder not found`

The core YAML folder does not exist yet. Run `expander.py` first:

```powershell
python expander\expander.py longitudinal/acc/acc_csc_001 --clean
```

`Storyboard template not found`

Check that the storyboard template exists:

```text
scenarios/general_scenarios/templates/storyboard/<feature_domain>/<functional>_storyboard.xosc
```

`Template file not found`

Check that the maneuver block exists:

```text
scenarios/general_scenarios/templates/maneuver_blocks/<feature_domain>/<functional>/<block_file>.xosc
```

`Unknown maneuver type`

The logical YAML contains a maneuver `type` that is not defined in `MANEUVER_MAP` inside `generated.py`.

## 13. Recommended Workflow

1. Run `expander.py` for one scenario.
2. Inspect the generated core YAML files.
3. Run `generated.py` for that same scenario.
4. Inspect the generated `.xosc` files.
5. Run a range or functional folder only after the single scenario is correct.
